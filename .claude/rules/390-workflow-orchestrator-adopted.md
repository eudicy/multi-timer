# Pre-Commit Workflow

Before every git commit (including TDD red/green/refactor commits), you MUST follow these steps in this exact order:

1.  **Run Unit Tests**: Ensure all tests pass.
    - Execute the project's unit test suite
    - Verify all tests pass with no failures
    - See project documentation for specific test commands

2.  **Create Commit**: Single commit with all changes.
    - Include code, tests, and memory bank updates in one commit
    - Use /commit skill to commit
